## Decisiones técnicas

- **DateOnly / TimeOnly**: Native Npgsql 10 types. Frontend sends ISO 8601 strings (YYYY-MM-DD for date, HH:mm for time).
- **Validations**: 
  - Sala y Expositor must belong to same Conferencia (returns SalaNotInConferencia / ExpositorNotInConferencia)
  - HoraInicio < HoraFin (InvalidTimeRange)
  - Fecha within Conferencia.FechaInicio..FechaFin (InvalidDateRange)
- **QrCodeUrl & EncuestaUrl**: Stored as text fields, not generated by backend (QR generated by Next.js mini-sitio).
- **Ownership**: Verified via Conferencia.UsuarioId ownership check.
- **DELETE**: Currently allows delete even with encuestas (TODO US-10/11: add encuesta check with respuestas).
- **SesionListItemDto**: Includes SalaNombre & ExpositorNombre for list display (loaded via FK navigation in EF).

## Cómo probar

```bash
cd backend && dotnet ef database update
dotnet run
```

Endpoints:
- `GET /api/dashboard/conferencias/{conferenciaId}/sesiones` — lista
- `POST /api/dashboard/conferencias/{conferenciaId}/sesiones` — crear
- `GET /api/dashboard/conferencias/{conferenciaId}/sesiones/{id}` — detalle
- `PUT /api/dashboard/conferencias/{conferenciaId}/sesiones/{id}` — editar
- `DELETE /api/dashboard/conferencias/{conferenciaId}/sesiones/{id}` — eliminar

Angular: `/congreso/:id/sesiones` con authGuard.
