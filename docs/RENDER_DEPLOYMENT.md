# Render Deployment Guide

## Overview

Deploy .NET backend + PostgreSQL to Render (Free tier).

**Render Free:**
- $0 base + $5 credits/month
- PostgreSQL 1 GB free
- Backend always-on (no spin-down)
- Sufficient for hobby projects

## Prerequisites

- Render account (free)
- GitHub repo connected
- docker-compose.yml + Dockerfile ready

## Step 1: Create PostgreSQL Database

1. [render.com](https://render.com) → **Dashboard**
2. Click **"New"** → **PostgreSQL**
3. Fill:
   - Name: `conference-db`
   - Database: `conference_db`
   - User: `postgres`
4. Click **Create Database**

Wait 1-2 min for database to be ready.

**Note:** Copy connection string:
```
postgresql://user:password@hostname:5432/database
```

## Step 2: Create Web Service (Backend)

1. **Dashboard** → Click **"New"** → **Web Service**
2. Select GitHub repo: `luchoespejo/ConferenceManager`
3. Configure:
   - **Name:** `conference-api`
   - **Branch:** `main`
   - **Root Directory:** `./backend`
   - **Runtime:** `Docker`
   - **Build Command:** (auto)
   - **Start Command:** (auto)
4. Click **Create Web Service**

Wait 5-10 min for first build.

## Step 3: Add Environment Variables (Secrets)

1. Web service → **Settings** → **Environment**
2. Add variables (copy from `.env.example`):

```
# Database (from PostgreSQL service)
ConnectionStrings__DefaultConnection=postgresql://user:password@hostname:5432/conference_db

# JWT (generate 32+ char random string)
Jwt__SecretKey=your-secret-key-here-minimum-32-characters

# Resend Email
Resend__ApiKey=re_your_api_key_here
Resend__FromAddress=noreply@tuplataforma.com
Email__UseFake=false

# URLs (after deployment, update with actual URLs)
Cors__AllowedOrigins__0=https://your-vercel-site.vercel.app
App__BaseUrl=https://conference-api.onrender.com
App__SiteUrl=https://your-vercel-site.vercel.app

# Environment
ASPNETCORE_ENVIRONMENT=Production
```

3. Click **Deploy**

## Step 4: Link PostgreSQL to Backend

1. Backend service → **Environment**
2. Under **Render Postgres** section:
   - Check if auto-linked
   - If not, manually add:
     ```
     DATABASE_URL=${{Postgres.DATABASE_URL}}
     ```

## Step 5: Verify Deployment

1. Backend service → **Logs**
   - Look for: `Application started`
   - If errors: check env vars

2. Get API URL:
   - **Settings** → **Domains**
   - Copy: `https://conference-api-xxxx.onrender.com`

3. Test:
   ```bash
   curl https://conference-api-xxxx.onrender.com/api/public/test-slug
   ```

## Step 6: Update Vercel with Backend URL

1. Vercel → Project **Settings**
2. **Environment Variables**
3. Update:
   ```
   NEXT_PUBLIC_API_URL=https://conference-api-xxxx.onrender.com
   ```
4. Redeploy site: `git push origin main`

## Secrets Management

### Generate JWT Secret

```bash
# Linux/Mac
openssl rand -base64 32

# Windows PowerShell
[Convert]::ToBase64String((Get-Random -InputObject (0..255) -Count 32))
```

### Resend API Key

1. [resend.com](https://resend.com) → **API Keys**
2. Copy key: `re_...`
3. Add to Render as: `Resend__ApiKey`

Use generated by Render automatically.

## Troubleshooting

### Build fails

Check logs:
```
Service → Logs tab
```

Common:
- Missing `Dockerfile` in root of backend
- Root directory wrong (should be `./backend`)

### Container won't start

```
"Application started" missing in logs
```

Check:
- `ASPNETCORE_ENVIRONMENT=Production`
- `ASPNETCORE_URLS=http://+:8080`
- All env vars set correctly
- Database connection string valid

### Database connection error

```
Npgsql.NpgsqlException: timeout expired
```

**Fix:**
1. Verify PostgreSQL service is running
2. Check `ConnectionStrings__DefaultConnection` matches PostgreSQL connection string
3. Test connection:
   ```bash
   psql postgresql://user:pass@host:5432/db
   ```

### API returns 500 error

Check Render logs for exceptions:
- Missing config
- Invalid JWT secret
- Email service misconfigured

## Custom Domain

1. Service → **Settings** → **Custom Domain**
2. Add: `api.tuplataforma.com`
3. Update DNS:
   ```
   CNAME: api.tuplataforma.com → onrender.com (provided)
   ```

## Auto-Deploy Configuration

1. Service → **Settings** → **Deploys**
2. **Auto-Deploy** = `on`
3. Branch: `main`

Now every push to main auto-triggers Render build.

## Backup & Restore

### Database Backups

Render auto-backs up daily. Manual backup:

```bash
# Via Render CLI
render pg:backups postgres-db

# Via psql
pg_dump $DATABASE_URL > backup.sql
pg_restore -d $DATABASE_URL < backup.sql
```

## EF Core Migrations

### Auto-apply on deploy

Add to Dockerfile:

```dockerfile
RUN dotnet ef database update -c AppDbContext || true
```

### Manual apply

```bash
dotnet ef database update -c AppDbContext
```
